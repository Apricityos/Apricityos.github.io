18铁人三项赛

题目描述

1.黑客攻击的第一个受害主机的网卡IP地址
2.黑客对URL的哪一个参数实施了SQL注入
3.第一个受害主机网站数据库的表前缀(加上下划线 例如abc_)
4.第一个受害主机网站数据库的名字

5.Joomla后台管理员的密码是多少
6.黑客第一次获得的php木马的密码是什么
7.黑客第二次上传php木马是什么时间
8.第二次上传的木马通过HTTP协议中的哪个头传递数据
9.内网主机的mysql用户名和请求连接的密码hash是多少(用户:密码hash)
10.php代理第一次被使用时最先连接了哪个IP地址
11.黑客第一次获取到当前目录下的文件列表的漏洞利用请求发生在什么时候
12.黑客在内网主机中添加的用户名和密码是多少
13.黑客从内网服务器中下载下来的文件名

wp

黑客攻击的第一个受害主机的网卡IP地址

wireshark抓包后过滤http，注意到ip为202.1.1.2对192.168.1.8不断地发包

用http && ip.src==202.1.1.2筛选源ip

注入参数

从大量数据中可以看出是进行了sql注入，且注入点是表单list

第一个受害主机网站数据库的表前缀

选择一条数据，进行tcp流跟踪

![屏幕截图 2025-03-25 163025](C:\Users\Seven\Pictures\Screenshots\屏幕截图 2025-03-25 163025.png)

在流中发现目标站点数据库抛出的错误，FROM ajtuc_ucm_history AS h

LEFT JOIN ajtuc_users，可以确定目标站点数据库表名前缀是ajtuc_

第一个受害主机网站数据库的名字

再次进行过滤(ip.addr == 192.168.1.8 || ip.addr == 202.1.1.2) && http

找到最后一次注入（根据time和传输信息）的payload利用在线平台进行解码，得到

Request URI Query Parameter […]: list[select]=(UPDATEXML(6315,CONCAT(0x2e,0x71717a7671,(SELECT MID((IFNULL(CAST(username AS CHAR),0x20)),1,22) FROM joomla.ajtuc_users ORDER BY id LIMI

可以确定数据库名为joomla

后台管理员的密码

去查看黑客的POST传参数据，或者去找登录页面

查看黑客POST传参的流量：ip.addr == 202.1.1.2 && http.request.method == POST

或者直接查找登录界面：ip.addr == 202.1.1.2 && http contains"login"

从数据包中可直接获取username和password

黑客第一次获得的php木马的密码

继续查看post流量

ip.addr == 202.1.1.2 && http.request.method == POST

得到key密码为zzz

黑客第二次上传php木马是什么时间

过滤kkkaaa.php文件ip.addr == 202.1.1.2 && http contains"kkkaaa.php"

出现三个传输参数，一般是上传文件

z0传命令，z1是上传文件的路径以及名字，z2是文件内容（16进制）

把每一个参数进行base64解码得：

z1：/var/www/html/joomla/tmp/footer.php可见传了一个php文件

z2十六进制解码后再进行解混淆，在结尾添加var_dump($j);var_dump($x);解混淆后发现有eval出现说明为第二次上传木马，在frame标头下成功找到arrival time

第二次上传的木马通过HTTP协议中的哪个头传递数据

前面可知木马文件是footer.php

http流中过滤该文件ip.addr == 202.1.1.2 && http contains"footer.php"

发现通过http协议中的referer头传递数据

内网主机的mysql用户名和请求连接的密码hash是多少

查看通过第一台服务器为跳板来连接的mysqlip.addr == 192.168.1.8 && mysql

出现response ok即为登录成功，查看上次登录信息得到正确的密码hash

php代理第一次被使用时最先连接了哪个IP地址

继续查看黑客的操作ip.src == 202.1.1.2 && ip.dst == 192.168.1.8 && http

发现多了一个php文件并且出现了connect连接通过时间得到第一次使用的ip地址

黑客第一次获取到当前目录下的文件列表的漏洞利用请求发生在什么时候

继续进行过滤(ip.addr == 192.168.1.8 || ip.addr == 202.1.1.2) && (http contains "dir" || http contains "ls")

可以看到黑客把192.168.1.8这台服务器当作跳板在192.168.2.20上执行了命令

发现一条是ls，一条是dir

验证ls，追踪tcp流

发现并未成功获取到文件列表

再验证dir，追踪tcp

成功执行dir命令，进而找到相应时间

黑客在内网主机中添加的用户名和密码是多少

通过上题得知内网主机指的是黑客通过192.168.1.8来攻击的192.168.2.20，黑客通过system来执行命令

过滤方式1通过system命令

ip.addr == 192.168.1.8 && http contains"system"

过滤方式2通过192.168.2.20的机器可以执行命令

ip.addr == 192.168.2.20 && http

可以看到黑客通过echo命令写入了sh.php的后门，查看后门数据

ip.addr == 192.168.1.8 && http contains"sh.php"

三个箭头分别代表上传、访问和执行

在一条数据中发现执行了net user添加用户的命令

后面的数据中发现添加的管理员用户为kaka

由以上得18:49:27.767754时，不存在kaka用户；18:50:42.908737时，kaka用户已成为管理员

在此期间的四个post请求中，第一个post解码得到

cd/d"C:\phpStudy\WWW\b2evolution\install\test\"&net user kaka kaka /add&echo [S]&cd&echo [E]

由net user kaka kaka /add得到用户名和密码均为kaka

黑客从内网服务器中下载下来的文件名

数据包最后解码得到C:phpStudyWWWb2evolutioninstalltestlsass.exe_180208_185247.dmp

可以确定下载的文件名
